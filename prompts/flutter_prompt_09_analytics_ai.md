Prompt 09 — Analytics Workbench, AI/Vision Insights, and Intelligence Studio

Role: As the principal data experience architect, define the comprehensive analytics, intelligence, and AI/vision workspaces for the RetailIQ Flutter application (Android + Web). You must translate all backend capabilities across `/api/v1/analytics/*`, `/api/v1/reports/*`, `/api/v1/ai/*`, `/api/v1/vision/*`, `/api/v1/nlp/*`, `/api/v1/decisions/*`, `/api/v1/events/*`, `/api/v1/market-intelligence/*`, and `/api/v1/developer/*` into precise Flutter modules. Reference the OpenAPI schemas to capture exact request fields, filters, response payloads, and streaming contracts.

Functional Domains
1. Analytics Explorer
   - Multi-section dashboard for revenue, inventory, customer, payments, and events analytics. Each widget must map to backend endpoints (`/api/v1/analytics/revenue`, `/api/v1/analytics/inventory`, etc.) with the ability to pivot by date range, store, category, channel.
   - Provide builder UI for custom reports referencing `/api/v1/reports/query`. Support drag-and-drop metrics/dimensions, filter chips, group-by selection, preview table, and chart toggles (line, bar, waterfall).
   - Implement scheduled report workflows: configure cadence, recipients, format (CSV, PDF) via `/api/v1/reports/schedules`. Show upcoming deliveries and failure alerts.
2. AI & Vision Insights
   - Surface computer vision outputs from `/api/v1/vision/shelf-analytics`, `/api/v1/vision/invoices`, `/api/v1/vision/loss-prevention`. Provide gallery viewer, bounding box overlays, confidence metrics, and annotations saved back via `/api/v1/vision/annotations`.
   - Build OCR verification flow for invoices with side-by-side raw image, extracted entities, and correction tools. Sync corrections to backend endpoints.
   - Provide AI/NLP console referencing `/api/v1/nlp/query` (deterministic template responses) and `/api/v1/decisions/recommendations`. Show context, version, and allow injection of scenario inputs.
3. Intelligence & Event Sensing
   - Implement event-aware forecasting timeline overlay with `/api/v1/events/calendar` and include manual event creation hitting `/api/v1/events`. Provide conflict detection (overlapping promotions) and highlight effect on forecast lines.
   - Integrate anomaly detection queue from `/api/v1/market-intelligence/anomalies` with severity, explanation, and resolution workflow.
   - Provide automation builder linking decisions to actions (e.g., auto PO, alert). Compose flows referencing `/api/v1/decisions/playbooks` endpoints.
4. Developer Analytics & API Usage
   - Create developer portal section (for developer role) listing registered apps, credentials, usage metrics from `/api/v1/developer/apps`, `/api/v1/developer/usage`. Provide key rotation, webhook delivery logs, and OAuth client settings.

Architecture & Data Flow
- Modules: `features/analytics`, `features/reports`, `features/ai`, `features/vision`, `features/intelligence`, `features/developer`. Each module exposes repositories (Dio client + caching), domain services (query builders, anomaly interpreters), and presentation (Riverpod/Bloc controllers).
- Data virtualization: use `SyncfusionDataGrid` or `DataTable3` for large tables on desktop; fallback to paginated cards on mobile.
- Provide query cache with normalized keys for repeated analytics requests. Implement offline mode limited to last synced datasets stored in Hive/Isar encrypted boxes.
- Build `ChartEngine` service to convert backend aggregates into chart series; support live updates via SSE/WebSocket when `/stream` endpoints exist.

UX & Interaction
- Responsive layout: desktop uses docking panes (filters left, canvas center, details right); tablet uses collapsible filter drawer; mobile uses stacked sheets.
- Provide accessible controls (keyboard shortcuts, focus order, semantics). Add tooltips for metrics referencing backend metadata (`description`, `formula`).
- Implement `InsightNarrator` widget summarizing key takeaways generated by backend `insight_text` fields.
- Allow bookmarking/saving of chart configurations via `/api/v1/analytics/views`. Persist layout per user.

Testing & Telemetry
- Unit tests for query builder serialization, chart engine, AI annotation store.
- Integration tests mocking Dio for report scheduling, anomaly resolution, developer key rotation.
- Analytics instrumentation: `report_saved`, `vision_annotation_submitted`, `nlp_query_run`, `developer_key_rotated`.

Deliverable: Provide file hierarchy, provider graph, widget trees for each major screen, state diagrams, caching strategy notes, streaming protocol summary, and tables enumerating all API contracts used (path, method, request schema, response schema). Ensure the blueprint is exhaustive enough for implementation teams to proceed without assumptions.
